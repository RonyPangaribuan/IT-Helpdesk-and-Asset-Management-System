# Screenshot Checklist

Real screenshots must be captured from the running DelDesk application. Do not use placeholder images or screenshots from another project.

Store completed screenshots in `docs/screenshots/` and update the README only for images that actually exist.

## Required Screenshots

- [ ] Landing page.
- [ ] Login page.
- [ ] Requester dashboard.
- [ ] Create ticket.
- [ ] Ticket detail with comments and attachments.
- [ ] Technician dashboard.
- [ ] Technician In Progress ticket.
- [ ] Resolve ticket form.
- [ ] Admin dashboard.
- [ ] Assignment workflow.
- [ ] Asset list.
- [ ] Asset detail.
- [ ] User management.
- [ ] Mobile navigation.
- [ ] GitHub Actions CI success.

## Capture Notes

- Use local/demo accounts only.
- Avoid exposing real credentials, production secrets, or private user data.
- Capture desktop and mobile where layout differences matter.
- Confirm each screenshot shows real application data generated by seeders or manual demo setup.
- Leave unchecked items as manual action until the image file exists.
